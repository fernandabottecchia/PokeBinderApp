Casos de Uso (POKÉBINDER) - Grupo X

  1. Caso de Uso: Listar Cartas Faltantes
  • Ator Principal: Colecionador/Jogador
  • Descrição: O Colecionador/Jogador deseja visualizar uma lista de cartas de Pokémon faltantes em sua coleção.
  • Fluxo Principal:
  1. O Colecionador/Jogador abre o aplicativo.
  2. Seleciona a opção "Mostrar Cartas Faltantes".
  3. Visualiza uma lista de cartas não cadastradas com informações como nome, preço e vendedor.
  • Fluxo Alternativo:
  • 3.1. Se não houver cartas faltantes, exibir uma mensagem informando que a coleção está completa.

  1. Caso de Uso: Pesquisar Cartas por Nome ou Número
  • Ator Principal: Colecionador/Jogador
  • Descrição: O Colecionador/Jogador deseja pesquisar cartas de Pokémon específicas por nome ou número de carta.
  • Fluxo Principal:
  1. O Colecionador/Jogador abre o aplicativo.
  2. Seleciona a opção "Pesquisar Cartas".
  3. Insere o nome ou número da carta desejada.
  4. O sistema exibe resultados correspondentes à pesquisa.
  • Fluxo Alternativo:
  • 4.1. Se não houver resultados correspondentes à pesquisa, o sistema informa que não foram encontradas cartas correspondentes.

  1. Caso de Uso: Avaliar Preço de Cartas
  • Ator Principal: Colecionador
  • Descrição: O colecionador deseja obter uma estimativa de preço para uma carta específica.
  • Fluxo Principal:
  1. O colecionador abre o aplicativo.
  2. Seleciona a opção "Avaliar Preço de Cartas".
  3. Insere o nome ou número da carta desejada.
  4. O sistema fornece uma estimativa de preço com base em dados agregados.
  • Fluxo Alternativo:
  • 4.1. Se a estimativa de preço não estiver disponível, o sistema informa que os dados não estão disponíveis.

  1. Caso de Uso: Recomendações de Cartas
  • Ator Principal: Colecionador
  • Descrição: O Colecionador deseja receber recomendações personalizadas de cartas com base em seus interesses de coleção.
  • Fluxo Principal:
  1. O Colecionador abre o aplicativo.
  2. Acessa a opção "Recomendações de Cartas".
  3. Define seus interesses de coleção, como tipos de cartas, edições preferidas, etc
  4. O sistema fornece recomendações de cartas alinhadas com os interesses definidos.
  • Fluxo Alternativo:
  • 4.1. Se o Colecionador não definir seus interesses, o sistema fornece recomendações gerais com base em tendências.

  1.  Caso de Uso: Cadastrar Preços de Cartas
  • Ator Principal: Vendedores de cartas, sites/lojas de cartas
  • Descrição: O Vendedor/Site/Loja deseja atualizar o preço de uma carta de Pokémon exibido no aplicativo.
  • Fluxo Principal:
  1. O Vendedor/Site/Loja abre o aplicativo.
  2. Seleciona a opção "Preços".
  3. Visualiza uma lista de cartas com informações como nome e preço.
  4. Procura por uma carta pelo nome, ano ou outra identificação.
  5. Seleciona a carta desejada e atualiza seu valor.
  • Fluxo Alternativo:
  •  3.1 Se a carta não for encontrada, é exibida uma mensagem informando a ausência do registro na base de dados.

  1. Caso de Uso: Adicionar Nova Carta à Coleção
  • Ator Principal: Colecionador
  • Descrição: O colecionador deseja adicionar uma nova carta à sua coleção.
  • Fluxo Principal:  
  1. O colecionador abre o aplicativo
  2. Acessa a opção "Gerenciar Coleção"
  3. Seleciona a ação "Adicionar Nova Carta à Coleção"
  4. O sistema exibe um formulário para inserir informações sobre a nova carta, incluindo nome, número, edição e quantidade desejada.
  5. O sistema adiciona a nova carta à coleção do colecionador e registra as informações inseridas.
  6. O sistema exibe uma confirmação de que a carta foi adicionada ou atualizada com sucesso.
  • Fluxo Alternativo:
  • 5.1.  Se a carta já existe, o sistema soma uma unidade da carta na coleção.

  1.  Caso de Uso: Remover Carta da Coleção
  • Ator Principal:  Colecionador
  • Descrição: O colecionador deseja remover uma carta de sua coleção.
  • Fluxo Principal:
  1. O colecionador abre o aplicativo.
  2. Acessa a opção "Gerenciar Coleção".
  3. Seleciona a ação "Remover Carta da Coleção".
  4. O sistema exibe a lista de cartas na coleção do colecionador.
  5. O colecionador seleciona a carta a ser removida.
  6. O sistema exibe uma mensagem de confirmação para garantir que o colecionador deseja remover a carta.
  7. O colecionador confirma a remoção, o sistema remove a carta da coleção do colecionador e atualiza a quantidade de cartas na coleção, se necessário.
  8. O sistema exibe uma confirmação de que a carta foi removida com sucesso.
  • Fluxo Alternativo:
  • 7.1. O colecionador cancela a remoção da carta.
  • 7.2. O sistema exibe uma confirmação de que a carta não foi removida.

  1.  Caso de Uso: Buscar Carta no Aplicativo
  • Ator Principal: Jogador
  • Descrição: O jogador deseja pesquisar cartas de Pokémon específicas dentro do aplicativo para obter informações detalhadas sobre essas cartas.
  • Fluxo Principal: O jogador abre o aplicativo.
  1. Acessa a opção "Buscar Carta".
  2. Insere o nome da carta, número de carta ou outros critérios de pesquisa.
  3. O sistema inicia a pesquisa com base nos critérios inseridos.
  4. O sistema exibe uma lista de cartas que correspondem aos critérios de pesquisa, incluindo informações detalhadas sobre cada carta, como nome, número, edição, tipo, raridade e habilidades.
  5. O jogador pode selecionar uma carta na lista para obter informações adicionais, como descrição, estatísticas de jogo e valor estimado.
  • Fluxo Alternativo:
  • 2.1 O jogador pode refinar a pesquisa adicionando mais critérios, como tipo de carta, edição específica, etc., e o sistema refaz a pesquisa com os novos critérios.
  • 2.2 O sistema oferece a opção de pesquisa avançada, na qual o jogador pode definir critérios adicionais, como ano da edição da carta, raridade, valor estimado, etc.
  • 4.1 Se não houver resultados correspondentes à pesquisa, o sistema informa que não foram encontradas cartas correspondentes.
  • 5.1 O jogador pode salvar as cartas pesquisadas em sua lista de cartas favoritas para acesso rápido no futuro.

  1. Caso de Uso: Criar Deck Personalizado
  • Ator Principal: Jogador
  • Descrição: O jogador deseja criar um deck personalizado, selecionando cartas de sua coleção para montar uma estratégia de jogo.
  • Fluxo Principal:
  1. O jogador abre o aplicativo.
  2. Acessa a opção "Criar Deck Personalizado".
  3. O sistema exibe a lista de cartas disponíveis na coleção do jogador.
  4. O jogador seleciona cartas da lista para adicionar ao deck. As cartas selecionadas podem ser organizadas em diferentes seções do deck, como cartas de Pokémon, cartas de treinador e cartas de energia.
  5. O sistema permite ao jogador visualizar o deck à medida que as cartas são adicionadas, mostrando a composição atual do deck.
  6. O jogador pode renomear o deck e salvar as configurações.
  7. O deck personalizado fica disponível para ser consultado posteriormente.
  • Fluxo Alternativo:
  • 4.1. O jogador não possui o número mínimo de cartas necessário para criar um deck.
  • 4.2. O sistema mostra uma mensagem informando que o usuário precisa registrar mais cartas para continuar e exibe um botão para abrir o menu de adicionar cartas.
